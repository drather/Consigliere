## 1.5 Environment Rule (CRITICAL)

이 프로젝트는 **ARM64 macOS** 환경에서 동작한다. 로컬에서 Python을 실행할 때 반드시 아래 규칙을 따른다.

### ❌ 절대 금지
```bash
python3 script.py        # 시스템 Python (x86_64) — 패키지 아키텍처 불일치로 반드시 실패
source .venv/bin/activate && pytest ...  # activate만으로는 부족 — arch 플래그 없으면 실패
```

### ✅ 올바른 실행 방법
모든 로컬 Python 실행은 반드시 아래 형식을 사용한다:
```bash
arch -arm64 .venv/bin/python3.12 script.py
arch -arm64 .venv/bin/python3.12 -m pytest tests/ -v
arch -arm64 .venv/bin/pip install <package>
```

> **이유:** `.venv`의 Python 바이너리는 universal binary(x86_64 + arm64)지만, pydantic-core 등 C 확장 패키지는 arm64 전용으로 컴파일되어 있다. `arch -arm64` 없이 실행하면 x86_64로 해석되어 `ImportError: incompatible architecture` 오류가 발생한다.

## 2. TDD (Test-Driven Development) — 필수

**모든 기능 구현은 TDD 방식으로 진행한다.**

> **"No test, no code."** 테스트 없이 로직을 먼저 구현하는 것은 금지.

| 단계 | 행동 |
|------|------|
| **Red** | 실패하는 테스트를 먼저 작성 (`tests/modules/{domain}/test_*.py`) |
| **Green** | 테스트를 통과시키는 최소한의 로직 구현 |
| **Refactor** | 코드 정리 (테스트는 계속 통과해야 함) |

### 테스트 구조
```
tests/
├── modules/
│   ├── career/
│   ├── real_estate/
│   └── finance/
├── api/
└── conftest.py
```

### 테스트 실행
```bash
# 전체 테스트
arch -arm64 .venv/bin/python3.12 -m pytest tests/ -v

# 특정 파일만
arch -arm64 .venv/bin/python3.12 -m pytest tests/test_career.py -v
```
