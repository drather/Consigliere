경로 상세 정보 — 어느 수준까지 보여줄까?

대시보드·리포트에서 출퇴근 경로를 표시하는 세 가지 방식

A

한 줄 요약

summary string만 저장, 파싱 단순

🚌 대중교통: 도보 5분 → 302번 버스 → 잠실역 → 2호선 → 삼성역 (59분)
🚗 자가용: 올림픽대로 → 테헤란로 (35분 · 15.0km)
🚶 도보: 가락로 → 송파대로 → 삼성역 (90분 · 5.0km)

장점

  • 구현 단순
  • 캐시 가볍다

단점

  • 구조화 불가
  • 아이콘/단계 분리 어려움
B

구조화 단계 목록 (추천)

legs[] JSON 저장, 대시보드에서 렌더링 가능

🚶 도보 출발지 → 302번 정류장 5분
가락로 방면 도보
🚌 302번 버스 잠실역 방향 12분 · 4정거장
가락시장 → 석촌 → 잠실 → 잠실역
🚇 2호선 잠실 → 삼성 8분 · 2정거장
잠실역 → 종합운동장 → 삼성역
🚶 도보 삼성역 → 목적지 3분

장점

  • 대시보드에서 단계별 렌더링
  • LLM이 환승 경로 정확히 인용

단점

  • DB 컬럼 JSON TEXT로 저장
  • 파싱 로직 추가
C

원시 API 응답 저장

전체 JSON blob 저장, 나중에 원하는 대로 파싱 가능

{"metaData":{"plan":{"itineraries":[{"totalTime":3540,"legs":[{"mode":"WALK","distance":400,"route":null...

장점

  • 정보 손실 없음
  • 미래 확장 자유도↑

단점

  • DB 매우 무거움
  • 모드별 스키마 달라 파싱 복잡
