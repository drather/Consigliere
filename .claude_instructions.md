## 1.5 Environment Rule (CRITICAL)

이 프로젝트는 **ARM64 macOS** 환경에서 동작한다. 로컬에서 Python을 실행할 때 반드시 아래 규칙을 따른다.

### ❌ 절대 금지
```bash
python3 script.py        # 시스템 Python (x86_64) — 패키지 아키텍처 불일치로 반드시 실패
source .venv/bin/activate && pytest ...  # activate만으로는 부족 — arch 플래그 없으면 실패
```

### ✅ 올바른 실행 방법
모든 로컬 Python 실행은 반드시 아래 형식을 사용한다:
```bash
arch -arm64 .venv/bin/python3.12 script.py
arch -arm64 .venv/bin/python3.12 -m pytest tests/ -v
arch -arm64 .venv/bin/pip install <package>
```

> **이유:** `.venv`의 Python 바이너리는 universal binary(x86_64 + arm64)지만, pydantic-core 등 C 확장 패키지는 arm64 전용으로 컴파일되어 있다. `arch -arm64` 없이 실행하면 x86_64로 해석되어 `ImportError: incompatible architecture` 오류가 발생한다.

## 2. Project Architecture Overview

- **API Server:** FastAPI (`src/main.py`), Docker container `consigliere_api` on port 8000.
- **Dashboard:** Streamlit (`src/dashboard/`), Docker container `consigliere_dashboard` on port 8501.
- **Automation:** n8n Docker container `consigliere_n8n` on port 5678.
- **Vector DB:** ChromaDB Docker container `consigliere_chromadb` on port 8001.
- **Domain Modules:** `src/modules/{domain}/` — currently `real_estate`, `finance`.
- **LLM Backend:** Swappable via `LLM_PROVIDER` env var. Currently `claude` (`claude-sonnet-4-6`).

## 3. SOLID Architecture Principles (Phase 4 Applied)

The following patterns are **mandatory** in all domain modules:

- **Single Responsibility:** Each class has ONE job. No God Classes.
  - `TourService` → 임장 기록/검색 전담
  - `InsightOrchestrator` → 멀티 에이전트 협업 전담
  - `RealEstatePresenter` → UI 포맷(Slack Block Kit) 전담
  - `RealEstateAgent` → 얇은 Facade. 서비스들을 조합만 함.
- **Open/Closed:** 새 에이전트는 `BaseAgent`를 상속하는 새 클래스로 추가. 기존 코드 수정 금지.
- **Zero Hardcoding:** API 코드, 금리, 지역코드 등 변경 가능한 값은 반드시 `config.yaml`에서 관리.
- **Composability:** 모든 서비스는 독립적으로 테스트 가능하고, 다른 모듈에서 재사용 가능하게 설계.

## 4. Standard Operating Procedure (SOP)

Every feature or refactoring follows this exact cycle:

| Phase                         | Actions                                                                                |
| ----------------------------- | -------------------------------------------------------------------------------------- |
| **Phase 0: Prep**             | `active_state.md` 업데이트 → Docker 상태 확인 → `git checkout -b feature/{name}`       |
| **Phase 1: Plan**             | `docs/features/{name}/spec.md` 작성 → `progress.md` 생성 → Commit                      |
| **Phase 2: Implement**        | **테스트 코드 먼저 작성 (TDD)** → 로직 구현 → `progress.md` 업데이트                   |
| **Phase 2.5: SOLID Review** ⭐ | SRP/OCP/DIP 체크 → Zero Hardcoding 확인 → 재사용성 검토 → 리팩토링 → 회귀 테스트       |
| **Phase 3: Document**         | `result.md` 작성 (walkthrough 포함) → `history.md` 업데이트 → system_snapshot 업데이트 |
| **Phase 4: Release**          | `master` 머지 → 테스트 → `git push origin master`                                      |

> ⭐ **Phase 2.5는 선택이 아닌 필수입니다.** 상세 체크리스트는 `docs/guidelines/workflow_sop.md` 참조.

## 5. TDD (Test-Driven Development) — 필수

**모든 기능 구현은 TDD 방식으로 진행한다.**

> **"No test, no code."** 테스트 없이 로직을 먼저 구현하는 것은 금지.

| 단계         | 행동                                                             |
| ------------ | ---------------------------------------------------------------- |
| **Red**      | 실패하는 테스트를 먼저 작성 (`tests/modules/{domain}/test_*.py`) |
| **Green**    | 테스트를 통과시키는 최소한의 로직 구현                           |
| **Refactor** | 코드 정리 (테스트는 계속 통과해야 함)                            |

### 테스트 구조
```
tests/
├── modules/
│   ├── career/
│   ├── real_estate/
│   └── finance/
├── api/
└── conftest.py
```

### 테스트 실행
```bash
# 전체 테스트
arch -arm64 .venv/bin/python3.12 -m pytest tests/ -v

# 특정 파일만
arch -arm64 .venv/bin/python3.12 -m pytest tests/test_career.py -v
```

## 7. Key Conventions

### 5.1 Language
- **모든 기술 문서 (Spec, Progress, Result, History):** 한글로 작성
- **Code & Comments:** English
- **User-facing report (Slack):** 한국어

### 5.2 LLM Usage
- **LLM Client:** `from core.llm import LLMClient` (Factory 패턴, 환경변수로 Gemini/Claude 전환)
- **Current Model:** `claude-sonnet-4-6` (`.env`의 `LLM_PROVIDER=claude`)
- **JSON 생성:** 항상 `llm.generate_json(prompt)` 사용. 절대 `eval()` 사용 금지.

### 5.3 Infrastructure (CRITICAL)
- **Environment:** ALWAYS activate the virtual environment before running any local commands: `source .venv/bin/activate`
- **Never** run Python/pip without Docker context or active `.venv` unless explicitly debugging.
- Docker 재빌드 필요 시: `docker compose up -d --build api`

### 5.4 n8n Workflows
- **Never** write n8n JSON from scratch. Use templates in `src/n8n/templates/`.
- Use `scripts/deploy_workflows.py` for deployment (idempotent, auto-activates).
- New workflows must be registered in `docs/workflows_registry.md`.
